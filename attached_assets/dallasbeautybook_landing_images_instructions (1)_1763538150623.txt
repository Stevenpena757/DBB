-------

  (Skincare jars, serums, towels, facial steamer, masks.)

Implementation in BusinessCard component (or equivalent):

```tsx
// Example: src/components/BusinessCard.tsx

const categoryImageMap: Record<string, string> = {
  "Med Spa": "/images/dallasbeautybook/cat-medspa.jpg",
  "Aesthetics Spa": "/images/dallasbeautybook/cat-medspa.jpg",
  "Medical Aesthetics": "/images/dallasbeautybook/cat-injectables.jpg",
  "Injectables": "/images/dallasbeautybook/cat-injectables.jpg",
  "Lashes": "/images/dallasbeautybook/cat-lashes.jpg",
  "Brows": "/images/dallasbeautybook/cat-brows.jpg",
  "Hair Salon": "/images/dallasbeautybook/cat-hair.jpg",
  "Nail Salon": "/images/dallasbeautybook/cat-nails.jpg",
  "Skin": "/images/dallasbeautybook/cat-skin.jpg",
};

function getBusinessImage(biz: Business) {
  if (biz.heroImageUrl) return biz.heroImageUrl;
  if (biz.category && categoryImageMap[biz.category]) {
    return categoryImageMap[biz.category];
  }
  return "/images/dallasbeautybook/cat-medspa.jpg";
}
```

Then in the JSX:

```tsx
<img
  src={getBusinessImage(business)}
  alt={`${business.name} – ${business.category ?? "beauty service"}`}
  className="h-40 w-full rounded-2xl object-cover"
  loading="lazy"
/>
```

This ensures:
- Featured / directory cards look engaging.
- Users see relevant imagery without businesses needing to upload assets immediately.

========================================================
7. MOBILE OPTIMIZATION
========================================================

- All <img> must:
  • Use className="w-full h-full object-cover" inside a fixed-height container OR
  • Use className="max-w-full h-auto" when inline.

- For mobile:
  • Stack hero image BELOW text or move image into a smaller card below CTAs.
  • Use loading="lazy" for all non-hero images to improve performance.
  • Keep the largest image above the fold limited to the hero only.

Example for mobile stacking:

- On small screens:
  • Show text + CTAs first.
  • Place hero image below in a 200–240px tall container.

========================================================
8. ACCESSIBILITY & ALT TEXT
========================================================

Decorative images (backgrounds, ambient art):
- Use alt="" so screen readers skip them.

Informational images (hero, category depictions, quiz teaser):
- Write alt text that:
  • Describes what is in the image (tools, textures, ambience).
  • Ties it to the page context (beauty services, planning, analytics).

Examples:
- "Beauty tools, skincare bottles and a jade roller on a neutral background"
- "Lash trays, wands and tweezers arranged neatly on a table"
- "Abstract layout of a laptop and charts showing business analytics"

Avoid:
- Faces
- Gender-specific language in alt (unless explicitly needed)
- Overly long alt text
