--------------------------------
0.1 Do NOT:
- Delete or rename existing pages, routes, or major components.
- Remove any existing CTAs, links, or sections.
- Modify existing backend API endpoints or database schemas.
- Break or refactor core flows such as Explore, Community, Beauty Book, Claims, Admin, or Auth.

0.2 You MAY:
- Update text copy.
- Adjust spacing, ordering, and layout of sections on the homepage.
- Add new sections and pages.
- Limit the number of items visually rendered (while keeping “View all” links).
- Add new read-only UI that surfaces existing data.

0.3 Frontend Safety:
- Do NOT modify Tailwind configuration files, design tokens, or global theme variables.
- Do NOT edit or drastically change global styles (e.g., global.css / index.css) beyond what already exists.
- For any new UI, rely on existing utility classes and patterns already used elsewhere in the app.

0.4 Component & Import Safety:
- Do NOT rename existing component files or imports.
- Do NOT move existing components into different folders.
- If you need new UI pieces, create NEW small components instead of refactoring existing ones.
- Keep current import paths and names intact to avoid breaking the app.

0.5 Responsiveness:
- Maintain existing mobile-first patterns and breakpoints.
- For new sections, use the same responsive patterns (e.g., grid-cols-1 md:grid-cols-3, mt-12 md:mt-16).
- Do NOT introduce new breakpoints or change overall mobile layout structure.
- Always verify that desktop, tablet, and mobile layouts remain usable and visually clean.
